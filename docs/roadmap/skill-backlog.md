# Skill-Backlog (Research-Struktur)

Der Begriff „Top 10" wird nicht erfunden oder hart codiert. Dieses Backlog
definiert die Struktur; die eigentliche Nischenrecherche ist eine separate,
datierte Research-Aufgabe (Rolle 6) mit echten Datenquellen.

## Backlog-Schema (je Eintrag verpflichtend)

| Feld | Bedeutung |
|---|---|
| id | SKB-xxx |
| datenquelle | z. B. KDP-Kategorie-Ranglisten, Keyword-Tools, Datum + URL |
| markt | amazon.de / .com / .es … |
| erhebungsdatum | ISO-Datum der Recherche |
| nische | Kategorie/Thema |
| nachfrageindikatoren | BSR-Bänder, Suchvolumen, Trend |
| wettbewerb | Titelzahl, Review-Dichte, Qualitätsniveau |
| formatEignung | Workbook / Paperback / E-Book / Mischformen |
| risiko | rechtlich, sensibel (Gesundheit/Kinder), Sättigung |
| skillVorschlag | Skill-Key + benötigte Safety-/Audience-Profile |
| prioritaet | P0–P3 mit Begründung |
